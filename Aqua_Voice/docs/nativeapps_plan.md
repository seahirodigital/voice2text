# Aqua Voice ネイティブアプリ化 実行計画

## 目的

`/Users/user/Library/CloudStorage/OneDrive-個人用/開発/Voice2Text/Aqua_Voice` のAqua Voiceを、ブラウザUI中心のMVPから、画面上部に常駐するネイティブアプリへ移行する。

現在のブラウザ版では、音声認識とGroq LLM整形は動作しているが、録音開始や操作のためにブラウザが前面へ出ることで、貼り付け先アプリの記憶が不安定になりやすい。ネイティブアプリ化により、入力中のアプリを邪魔せず、録音状態と変換結果を見ながら、整形済みテキストを対象アプリへ自動貼り付けできる状態を目指す。

## 現状の課題

主要ファイルは `/Users/user/Library/CloudStorage/OneDrive-個人用/開発/Voice2Text/Aqua_Voice/aqua_voice_app.py` である。

現在は次の機能が1ファイル内にまとまっている。

- ローカルWeb UI提供
- グローバルショートカット監視
- マイク録音
- Groq音声認識
- Groq LLM整形
- macOS / Windows 自動貼り付け
- 設定保存

この構成では、音声処理自体は成立している一方で、UIがブラウザであることにより、次の問題が起きやすい。

- ブラウザ操作で録音開始すると、貼り付け先がブラウザとして記憶される。
- 変換結果を見ながら操作したい場合、ブラウザが入力先アプリの前面状態を奪う。
- 自動貼り付け対象と操作画面が分離しておらず、体験が不安定になる。
- ネイティブアプリのような常駐感、即時性、視認性が不足している。

## 基本方針

UIをブラウザからネイティブ常駐UIへ移行する。

ネイティブUIは、macOSとWindowsの両方で動かすため、`PySide6` を第一候補とする。過去にTkinterでクラッシュした履歴があるため、Tkinterは採用しない。

アプリは画面上部に小さく表示し、必要な情報だけを常時確認できるようにする。イメージとしては、音声入力中に邪魔にならない細いステータスバー型UIである。

## 目標UI

画面上部に、常に最前面または必要時だけ前面表示されるネイティブUIを出す。

表示する情報は次の通り。

- 録音中 / 待機中 / 処理中 / 貼り付け完了 / エラー
- 入力音量メーター
- リアルタイム音声認識結果
- Groq LLM整形結果
- 貼り付け先アプリ名
- 音声認識レイテンシ
- LLM整形レイテンシ
- Groq APIキー設定状態

UI上に最低限必要な操作を置く。

- 録音開始
- 停止して貼り付け
- ロック録音切替
- 再貼り付け
- クリア
- 設定
- 終了

## アーキテクチャ分離計画

`/Users/user/Library/CloudStorage/OneDrive-個人用/開発/Voice2Text/Aqua_Voice/aqua_voice_app.py` にまとまっている処理を、段階的に分離する。

### 1. 音声録音エンジン

既存の `AudioRecorder` 相当を独立させる。

責務:

- マイク入力開始
- マイク入力停止
- 音声バッファ保持
- 入力レベル算出

理由:

UIがWebでもネイティブでも、録音処理は共通で使うため。

### 2. Groq処理エンジン

既存の `GroqClient` 相当を独立させる。

責務:

- Groq音声認識API呼び出し
- Groq LLM整形API呼び出し
- APIキー検証
- レイテンシ計測

理由:

音声認識とLLM整形は現在うまく動いているため、UI変更の影響を受けない部品として守る。

### 3. 自動貼り付けエンジン

既存の `AutoPaste` 相当を独立させる。

責務:

- 録音開始時の貼り付け先アプリ記憶
- ネイティブUI自身を貼り付け先から除外
- 対象アプリの再アクティブ化
- クリップボード書き込み
- `Command+V` / `Ctrl+V` 送信

理由:

今回の中心課題は貼り付け先の安定化であり、UI自身を入力先として誤認しない設計が必要なため。

### 4. アプリ状態管理

既存の `RuntimeState` 相当を残しつつ、UIとバックエンド処理の間に置く。

責務:

- 録音状態
- 処理状態
- エラー状態
- リアルタイム認識結果
- 整形結果
- 貼り付け先アプリ
- 設定値

理由:

ネイティブUIは状態を描画するだけに近づけ、音声処理の複雑さをUIへ持ち込まないため。

### 5. ネイティブUI

`PySide6` で新規作成する。

責務:

- 上部オーバーレイ表示
- 状態表示
- 操作ボタン
- 設定画面
- 辞書設定画面

理由:

ブラウザを起動しないことで、貼り付け先アプリの前面状態を守るため。

## 自動貼り付け仕様

### 共通仕様

録音開始時に、現在の入力先アプリを記憶する。

ただし、ネイティブUI自身が前面にいる場合は、自分自身を貼り付け先として記憶しない。直前に使っていた非Aqua Voiceアプリを貼り付け先として維持する。

貼り付け時は次の順で実行する。

1. 整形済みテキストを確定する。
2. 記憶済みの対象アプリを前面に戻す。
3. クリップボードへ整形済みテキストを書き込む。
4. 対象アプリへ貼り付けショートカットを送信する。
5. UIへ貼り付け成功または失敗を表示する。

### macOS

macOSでは次を使う。

- `osascript`
- `System Events`
- `pbcopy`
- `Command+V`

必要な権限:

- マイク
- アクセシビリティ
- 入力監視

対象アプリの識別には、可能な限りbundle identifierを使う。

### Windows

Windowsでは次を使う。

- `ctypes`
- `GetForegroundWindow`
- `GetWindowTextW`
- `SetForegroundWindow`
- `clip`
- `Ctrl+V`

対象アプリの識別には、ウィンドウハンドルを使う。ただし、ウィンドウハンドルはアプリ再起動で無効化されることがあるため、貼り付け失敗時にはエラーとしてUIへ表示する。

## ショートカット仕様

既存仕様を維持する。

- `Option` 長押し中だけ録音
- `Option` ダブルタップでロック録音開始
- ロック録音中にもう一度ダブルタップで停止して貼り付け
- UIの停止ボタンでも停止して貼り付け

Windowsでは、設定により `ctrl`、`shift`、`f8` などへ変更できる状態を維持する。

## 設定画面

ブラウザ版で持っていた設定をネイティブUIへ移植する。

設定項目:

- Groq APIキー
- 音声認識モデル
- LLMモデル
- 言語
- 送信間隔
- サンプルレート
- ショートカットキー
- 辞書設定
- 自動貼り付けON/OFF

設定保存先は既存と同じ `/Users/user/Library/CloudStorage/OneDrive-個人用/開発/Voice2Text/Aqua_Voice/aqua_voice_config.json` を使う。

## リスクと対策

### OS権限で貼り付けできない

対策:

起動時またはエラー発生時に、必要な権限をUIへ表示する。

### UI自身が貼り付け先として記憶される

対策:

Aqua Voice自身のウィンドウ、プロセス名、bundle identifierを除外する。

### Windowsで対象ウィンドウを前面化できない

対策:

`SetForegroundWindow` が失敗した場合は、クリップボードへのコピー完了を表示し、手動貼り付け可能な状態を保証する。

### PySide6依存が重い

対策:

まずMVPとして導入し、起動速度や配布サイズが問題になった段階で、別のネイティブ技術を検討する。

- 必要な権限不足時に、原因がUIまたはエラー表示で分かる。
