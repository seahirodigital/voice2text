# プロダクト最終仕様書：Moonshine Local Transcriber (Phase 1)

## 1. プロダクトコンセプト
*   **コア価値**: 「軽量・高速・プライバシー」。Windowsローカル環境で、Moonshine ASRを用いた低遅延な文字起こしを実現する。
*   **モットー**: 思考を妨げないレスポンス速度と、モダンな操作感。

## 2. システムアーキテクチャ
*   **フロントエンド**: React (Vite), Tailwind CSS, shadcn/ui
    *   モダンで洗練されたUI/UX、高速なホットリロード。
*   **バックエンド**: Python 3.10+ (FastAPI), WebSocket
    *   リアルタイムな音声ストリーミングとテキストデータの双方向通信。
*   **推論エンジン**: [Moonshine](https://github.com/moonshine-ai/moonshine)
    *   ONNXまたはPyTorchによるローカル高速推論。
*   **話者識別 (Diarization)**: 
    *   Phase 1実装：2〜3人の話者を特徴量ベースで識別し、ラベル（話者A, B, C）を付与。

## 3. ストレージ・環境管理仕様
開発機と実利用機（ノートPC）の使い分け、およびOneDriveの同期容量を考慮した設計。

*   **OneDrive管理（ソースコード・設定）**:
    *   Reactソースコード、FastAPIロジック、UIデザイン資産。
    *   `config.json`（各環境のパス設定、Phase 2用のAPI設定）。
*   **ローカルストレージ分離（重量級ファイル）**:
    *   **モデルファイル**: `models/`（Moonshineの学習済みモデル：数GB）。
    *   **実行環境**: `.venv/`, `node_modules/`（ライブラリ群）。
    *   **音声データ**: `temp_recordings/`（録音されたwavファイル）。
*   **デプロイ/移行**:
    *   `setup.bat` を同梱し、各PCのローカル環境へモデルのダウンロードと環境構築を自動で行う仕様。

## 4. 主要機能（Phase 1）

### 4.1 録音・デバイス制御
*   **デバイス認識**: Windowsに接続されたUSBマイク、外部オーディオインターフェースを自動認識・選択。
*   **録音コントロール**:
    *   [録音開始]：リアルタイム推論開始。
    *   [一時停止]：認識を中断し、セッションを維持。
    *   [停止]：録音終了、音声ファイルを保存。
*   **視聴機能**: 録音した音源の再生・視聴停止・削除。

### 4.2 タイムライン型・リアルタイム文字起こし
*   **ストリーミングUI**:
    *   音声入力に合わせ、リアルタイムに波形（Visualizer）を表示。
    *   **3列構成のタイムライン**: `[経過時間] | [話者ラベル] | [編集可能なテキスト]`
    *   話者が切り替わるごとに自動で改行し、チャット形式で表示。
*   **インライン編集**:
    *   生成されたテキストはその場でユーザーが自由に追記・修正可能。
*   **全文コピー**: 
    *   「[00:15] 話者A: 内容...」の形式を維持したまま、ワンクリックでクリップボードへコピー。

### 4.3 Phase 2 準備機能（設定項目のみ）
*   **AI API連携設定**:
    *   将来的なテキスト整形（LLM）用。
    *   OpenAI / Anthropic等のAPIキー入力欄、モデル選択プロンプト設定UIを実装（Phase 1では保存のみ）。

## 5. UI/UX デザイン仕様
*   **デザインスタイル**: 今風の「モダン・ダーク/ライトモード」。
    *   **shadcn/ui** をベースとしたクリーンなコンポーネント。
    *   **Framer Motion** による滑らかなリストの追加アニメーション。
*   **画面構成**:
    1.  **サイドバー**: 履歴管理、設定（API設定、マイク選択）。
    2.  **メインパネル**: 
        *   上部：音声波形ビジュアライザー（リアルタイム）。
        *   中央：自動スクロール対応のタイムライン・エディタ。
        *   下部：録音/停止のフローティング操作パネル。

## 6. 機能要件
*   **パフォーマンス**: CPU環境でもストレスなく動作する軽量モデルの選定。
*   **オフライン動作**: API設定を除き、すべての機能がインターネット未接続で動作すること。
*   **拡張性**: 将来の話者分離精度の向上、日本語を見越したモジュール設計。
